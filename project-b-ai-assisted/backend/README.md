# Backend — URL Shortener API

FastAPI + SQLite. Full interactive API reference (request/response schemas,
try-it-out) is auto-generated by FastAPI at **`/docs`** (Swagger UI) and
`/redoc` once the server is running — that's the source of truth for request
bodies and validation rules. This file is a short summary plus the setup
commands.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows; use `source .venv/bin/activate` on macOS/Linux
pip install -r requirements.txt
cp .env.example .env          # then set a real API_KEY
uvicorn app.main:app --reload
```

Server starts on `http://localhost:8000`. It fails fast at startup if
`API_KEY` isn't set (see `app/config.py`) — copy `.env.example` to `.env`
first, or the process exits with a clear error instead of a confusing 401
on the first request.

Run the tests (from `project-b-ai-assisted/`, with the venv above active):

```bash
pytest tests -q
```

## Authentication

Endpoints that create or read data owned by the key holder require an
`X-API-Key` header matching the `API_KEY` environment variable, compared
with `secrets.compare_digest` to avoid a timing side-channel. The redirect
endpoint is intentionally public — anyone must be able to follow a
shortened link without credentials; that's the asymmetry the challenge
spec asks for. See `docs/architecture.md` for the full rationale.

## Endpoints

| Method | Path | Auth | Purpose |
|---|---|---|---|
| `POST` | `/api/shorten` | `X-API-Key` | Create a short code for a long URL. Returns `201` for a new code, `200` if the same URL was already shortened (dedup). |
| `GET` | `/{short_code}` | Public | Resolve a short code, increment its click count, and `307` redirect to the original URL. `404` if unknown, `410` if past `expires_at`. |
| `GET` | `/api/stats/{short_code}` | `X-API-Key` | Return click count and metadata for one short code. `404` if unknown. |
| `GET` | `/api/urls` | `X-API-Key` | List every short code created, newest first — powers the frontend dashboard. |
| `GET` | `/health` | Public | Liveness check, `{"status": "ok"}`. |

Every error response uses FastAPI's default `{"detail": "<message>"}` shape.
Full request/response JSON examples are in
[`../docs/architecture.md`](../docs/architecture.md#endpoints); the
authoritative, always-current schema is `/docs`.
